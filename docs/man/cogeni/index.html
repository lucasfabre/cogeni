cogeni(1)

NAME​

cogeni - Generate reproducible artifacts from code and specs

SYNOPSIS​

cogeni [FILES...] cogeni ast FILE [--lang LANG] cogeni clean [FILES...] cogeni diff [FILES...] cogeni run [SCRIPT.lua] [-e CODE] cogeni completion [bash|zsh|fish|powershell] cogeni show dependency-graph [ENTRY.lua] cogeni repl cogeni config show

DESCRIPTION​

cogeni is a programmable runtime for generating and synchronizing derived artifacts from source code and machine-readable specs. It uses Tree-sitter for AST parsing and JQ for declarative data querying. It features parallel execution, automatic dependency tracking, and atomic filesystem commits.

CORE WORKFLOW​

The tool operates in three distinct phases:

  1. Extraction Scans the file for <cogeni> blocks. It identifies and strips comment prefixes (e.g., "# ", "// ", "-- ") from each line within the block to assemble a valid Lua script.
  2. Execution Runs the assembled script in a specialized Lua environment. The script can parse the host file (or others) using cogeni.read_ast and capture output using the write(id, content) global function.
  3. Replacement Locates <generated by="cogeni" id="ID"> blocks and replaces their entire content with the strings registered via write. If the new content matches the existing content, the file remains untouched.

COMMANDS​

[default] Processes the provided file(s) for generation blocks and commits changes to disk. If no files are provided, it looks for cogeni.lua in the current directory. ast \fIFILE\fR Dumps the Tree-sitter AST of the specified file as JSON. clean [\fIFILES\fR...] Empties all <generated> blocks in the specified files. Useful for resetting a project state. diff [\fIFILES\fR...] Performs a dry-run of the generation and shows a unified diff of the changes that would be made. run [\fISCRIPT.lua\fR] Executes a Lua script. Useful for project-wide orchestration. If no script is provided, it looks for cogeni.lua. completion [\fIbash|zsh|fish|powershell\fR] Generates shell completion scripts for Bash, Zsh, Fish, and PowerShell. show dependency-graph [\fIENTRY.lua\fR] Visualizes the execution and file dependency tree discovered during processing. config show Displays the current configuration, including grammar mappings and custom sources.

TAGS​

<cogeni> ... </cogeni> Container for the generator script. <generated by="cogeni" id="ID"> ... </cogeni> Target for code injection. The closing tag is </cogeni>.

CONFIGURATION​

cogeni looks for config.yaml in standard XDG locations (e.g., ~/.config/cogeni/).

Example configuration: grammar: location: /path/to/grammars mapping: sources: sql: url: https://github.com/DerekStride/tree-sitter-sql branch: gh-pages

EXAMPLES​

Generate artifacts for a Python file: cogeni models.py See what changes would be made: cogeni diff . Orchestrate a whole project: cogeni run cogeni.lua

TIPS​

Use the repl command to experiment with JQ queries and Lua scripts in real-time. Combine cogeni ast with external tools like grep or jq to quickly find the node types you need to target.
